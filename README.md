# Spring Boot REST Stub API

Этот проект представляет собой простую заглушку (stub) REST API, разработанную с использованием Spring Boot и Apache Maven. Он предназначен для тестирования и имитации ответов сервера, включая задержку и обработку входящих данных.

## Возможности

*   **GET /api/data**: Возвращает статичный JSON-объект.
*   **POST /api/data**: Принимает JSON с `login` и `password`, возвращает их обратно вместе с текущей датой.
*   Имитация задержки ответа (1-2 секунды) для каждого запроса.
*   Базовая валидация для POST-запроса: возвращает `400 Bad Request`, если `login` или `password` отсутствуют.
*   Порядок полей в JSON-ответе гарантирован (`login`, `password`, `date`).

## Требования

*   Java Development Kit (JDK) 17 или выше
*   Apache Maven (встроен в проект как `mvnw`)

## Запуск приложения

1.  **Клонируйте репозиторий** (или перейдите в папку вашего проекта, если он уже у вас):
    ```bash
    cd Stub
    ```

2.  **Соберите проект**:
    Используйте встроенный Maven Wrapper для сборки проекта. Это создаст исполняемый JAR-файл в директории `target/`.
    ```bash
    ./mvnw clean package
    ```

3.  **Запустите приложение**:
    ```bash
    java -jar target/Stub-0.0.1-SNAPSHOT.jar
    ```
    Приложение запустится на порту `8080` (по умолчанию).

## API Эндпоинты

Все запросы обрабатываются по базовому пути `/api/data`.

### GET /api/data

*   **Описание**: Возвращает статичную информацию.
*   **Метод**: `GET`
*   **Пример ответа (200 OK)**:
    ```json
    {
        "login": "Login1",
        "status": "ok"
    }
    ```

### POST /api/data

*   **Описание**: Принимает учетные данные и возвращает их с текущей датой.
*   **Метод**: `POST`
*   **Заголовок**: `Content-Type: application/json`
*   **Пример тела запроса**:
    ```json
    {
        "login": "user_name",
        "password": "user_password"
    }
    ```
*   **Пример ответа (200 OK)**:
    ```json
    {
        "login": "user_name",
        "password": "user_password",
        "date": "2023-10-27 10:30:00"
    }
    ```
*   **Пример ответа (400 Bad Request)**:
    Если `login` или `password` отсутствуют в теле запроса.
    ```json
    {
        "error": "Login and password are required"
    }
    ```
